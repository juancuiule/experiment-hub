## Findings considered and rejected

So nobody re-audits these:

- **Unguarded debug panels exposing participant data**: already fixed —
  gated by `process.env.NODE_ENV === 'development'` in
  `apps/frontend/app/(experiments-layout)/experiments/[slug]/page.tsx:40,53`
  and `apps/frontend/src/Screen.tsx:66`. The CLAUDE.md claim saying
  otherwise is stale (plan 001 corrects it).
- **`selectStartNode` running before `validateExperiment`** (prior review
  Finding 3): fixed — `page.tsx:27` validates and early-returns before
  line 35 selects the start node.
- **Linear scans of nodes/edges per traversal step** (prior review
  Finding 2): fixed — `packages/engine/flow/graph.ts` builds an
  `ExperimentIndex` of Maps.
- **`shuffles.ts` hand-rolled component recursion / `shouldAutoTraverse`
  string array / `timing.ts` re-implementing state walking** (prior review
  Findings 1, 8, 9): all fixed — shuffles uses the `flatMap` walker,
  traverse uses the typed `isAutoTraverseNode` predicate, timing reuses
  `getLeafState`.
- **Answer-piping XSS via react-markdown**: not exploitable as evidenced —
  react-markdown escapes raw HTML by default (no rehype-raw in use,
  `RichText.tsx`), and piped answers render only in the authoring
  participant's own session. Revisit only if responses are ever shown to
  OTHER participants or an HTML plugin is added.
- **Query-param-driven start/condition selection as a vulnerability**:
  by design — it is the documented condition-assignment mechanism. Study
  integrity against URL tampering is a methodology concern for the
  researcher, not a code defect.
- **SSR hydration mismatch from randomization**: not real — the flow starts
  client-side in `Experiment.tsx`'s `useEffect`; shuffled content is never
  server-rendered.
- **Code-splitting the 12 response components**: not worth it now — adds
  Suspense waterfalls per screen for ~tens of KB on an app whose screens
  are sequential form pages; revisit if bundle analysis shows real cost.
- **Husky/pre-commit hooks & prettier --check in CI**: deliberately left as
  maintainer preference; CI lint/typecheck (plan 007) is the enforcement
  point.
- **Rewriting `deepMerge`'s internal `as any` recursion casts**
  (`packages/engine/flow/context.ts:9`): cosmetic gain, engine risk;
  callers are typed via plan 010's helper instead.
- **Moving `iterateLoops` out of `fields.ts`**: prior-review suggestion no
  longer applies as stated — it already lives in `fields.ts` and is
  imported by `screen-bindings.ts`/`screen-schema.ts`; fragmentation
  concern is documentation-level at most.
- **Direction items** (offered, not selected for planning this round):
  persistence backend for `send()` (the pre-study blocker), session resume
  (Zustand persist + restore), runtime experiment loading / authoring UI
  (incl. the fate of the `app/nodes` prototype), exporting `context.timings`
  response-time data to researchers.
